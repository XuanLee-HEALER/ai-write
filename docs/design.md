session流程

输入prompt -> 结合系统提示词组成请求参数
参数 -> 请求
请求 -> 失败，重试，重试失败多次退出循环（session退出）
重试成功回到1
请求 -> 响应
响应 -> tool，工具调用失败（重试），成功就组合参数继续调用请求
响应 -> end，session的总结（如果遇到异常，不end，需要做兜底，比如上限20轮）

session内有上下文统计（输入/输出/最大输出限制，上下文限制，触发压缩）

上层业务

一句话：基于deepseek的 ai和人协作进行写作的一种工作机制，暂时不考虑UI表现

工具会直接代码实现 文件管理（文件系统的管理，文件夹创建/修改/删除等），文本编辑（读/写/编辑/diff/find等），搜索（利用第三方服务mcp）
需要支持skill配置，系统提示词动态插入、修改

写作的时候，无论是什么主题的写作，都不能直接给ai一个题目，然后等它完成所有事情，一定是有参与感的，所以整个ai的过程要清晰，要可视化，这是需要在UI层面考虑的事情，但是底层机制需要提供这种能力，还需要支持人去修改一些内容

纯文字写作是比较基础的，还需要支持markdown，html（富文本），方便转pdf打印。写作成果的版权是AI（DeepSeek）和人类共有，每个文件要带有一个签名（deepseek模型id+人工签名）

文章的组织只要分两层，即主题和文章，主题下的文章的组织顺序通常是阅读顺序，即目录树结构，关系可以通过主题下做的索引来支持。主题直接对应一个目录，文章直接对应文件
